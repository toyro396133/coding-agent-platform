feat: Implement Asynchronous Loop and Clarification Panel for Tasks

This PR introduces the final major features requested: an asynchronous
agent loop structure within the Orchestrator, allowing it to define
subTasks in a JSONB field and iterate through them one by one. It
now properly pauses and switches to "awaiting_user_input" when
`[ASK_USER]` is yielded by the LLM.

To support this visually, the UI has been upgraded with a new
TaskSubTasksPanel component embedded in the chat view. Users can
see the sub-tasks accordion style, watch the progress, and answer
clarification questions directly, resuming the loop automatically.
